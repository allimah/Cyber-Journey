# الأسبوع الرابع: أساسيات Linux - الجزء الثاني

هذا الأسبوع يركز على المفاهيم الأكثر تقدمًا مثل صلاحيات الملفات وإدارة الحزم.

## 1. صلاحيات الملفات (File Permissions)

كل ملف ومجلد في Linux يمتلك صلاحيات تحدد من يمكنه الوصول إليه وكيف.

### فهم خرج `ls -l`

عند تنفيذ `ls -l`، يظهر خرج مثل `-rwxr-xr--`. يتم تقسيمه كالتالي:

| الجزء | المعنى |
| :--- | :--- |
| `-` | **نوع الملف**: `-` للملف العادي، `d` للمجلد. |
| `rwx` | **صلاحيات المالك (User)**: قراءة (r)، كتابة (w)، تنفيذ (x). |
| `r-x` | **صلاحيات المجموعة (Group)**: قراءة (r)، تنفيذ (x). |
| `r--` | **صلاحيات الآخرين (Others)**: قراءة (r) فقط. |

### الأمر `chmod`

يستخدم الأمر `chmod` (Change Mode) لتغيير هذه الصلاحيات.

| الأمر | الوظيفة |
| :--- | :--- |
| `chmod +x script.sh` | يضيف صلاحية التنفيذ (`x`) للجميع (المالك، المجموعة، الآخرون). |
| `chmod -w script.sh` | يزيل صلاحية الكتابة (`w`) من الجميع. |
| `chmod u=rwx,g=r,o=` | يحدد الصلاحيات بدقة: المالك يحصل على كل شيء، المجموعة تحصل على القراءة فقط، والآخرون لا يحصلون على أي صلاحيات. |
